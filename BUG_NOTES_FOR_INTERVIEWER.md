Sistem Pelaporan Gangguan TIK

Project ini telah memiliki fitur dasar, namun masih terdapat beberapa masalah pada logika, validasi, akses data, pencarian/filter, pagination, dan/atau lampiran file. 
1. tersedia akun staff dan user
2. tersedia beberapa sample ticket dari beberapa user
3. kandidat diharapkan menguji aplikasi dari perspektif role yang berbeda
4. kandidat diharapkan memeriksa fitur utama seperti form, akses data, filter, pagination, dan lampiran

Tugas Anda adalah:

1. menjalankan project,
2. mengidentifikasi masalah yang ada,
3. menjelaskan penyebab masalah tersebut,
4. memperbaiki masalah yang ditemukan,
5. memastikan aplikasi dapat berjalan dengan lebih baik setelah perbaikan.

Ketentuan Pengerjaan:

1. Gunakan project yang diberikan sebagai dasar.
2. Anda diperbolehkan memperbaiki file yang diperlukan.
3. Anda diperbolehkan menambah validasi, perbaikan query, pembatasan akses, atau penyesuaian kecil lain jika memang dibutuhkan.
4. Fokus utama adalah analisis dan perbaikan, bukan membuat ulang project dari nol.
5. Tidak perlu menambahkan fitur besar baru di luar kebutuhan yang ada, kecuali memang diperlukan untuk menyelesaikan masalah.

Instruksi Pengumpulan via Git

Hasil tugas dikumpulkan melalui repository Git milik masing-masing kandidat.

Langkah pengumpulan:

1. Clone repository starter yang diberikan panitia.
2. Jalankan project dan kerjakan perbaikannya.
3. Buat repository baru di akun GitHub pribadi Anda.
4. Hubungkan project lokal Anda ke repository baru tersebut.
5. Lakukan commit secara bertahap dengan pesan commit yang jelas.
6. Push seluruh hasil pekerjaan ke repository Anda.
7. Kirimkan link repository sebelum batas waktu (1 April 2026 23:59).
